librdesc

Introduction

librdesc is a table-driven recursive descent parsing library written in C99. librdesc features:

  • Table-Driven Parsing: Parser logic is generic and can be controlled at runtime without recompiling via the struct rdesc_cfg table.
  • Pump Mechanism: Parsing is driven by rdesc_pump(), a state machine that allows for token by token feeding.

API Overview

  1. Parser (rdesc.h): The engine that consumes the CFG and Input. It defines token and nonterminal structs and functions for bootstrapping the parser.
  2. Tree macros (cst_macros.h): Macros enabling access to fields of parsed CST.
  3. CFG (cfg.h): Defines the CFG struct for rules. This usually populated with static constant data via macros.
  4. Rule macros (bnf_macros.h): A set of macros to facilitate definning context free grammars. See examples below for usage.
  5. Optional features (util.h): Development and debugging utilities.

Quick Start Guide

To use librdesc, you typically follow these steps:

  1. Define the identifiers of tokens and nonterminals (e.g. create enums for your language symbols)
  2. Define your grammar, and its symbol table. You may use bnf_macros.h to create the rules table.
  3. Initialize:
    struct rdesc_cfg cfg;
    rdesc_cfg_init(&cfg, ...); // Load your rules
    struct rdesc parser;
    rdesc_init(&parser, &cfg, 8);
    // ^^^ 8 is the size your grammar's semantic information
